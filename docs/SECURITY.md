# 安全設計

Light Shelf AI は「小さい専門AIを必要な時だけ起こす」アプリですが、AIに直接PC操作をさせません。  
v1 の安全設計は Codex 風に、サンドボックス、承認、監査ログ、最小権限を軽量化して入れています。

## 基本ルール

- AIは提案だけを作る。
- 実操作は executor 側だけが行う。
- executor は AI の出力をもう一度検査する。
- ユーザー確認前にファイル変更しない。
- 削除、購入、ログイン、個人情報送信、権限変更、外部送信は安全レビュー対象にする。
- 秘密情報の表示、全削除、破壊的 git 操作はローカルポリシーで止める。
- 各AI定義は `safety.can_execute=false` を持つ。

## 守る層

```text
ユーザー入力
  ↓
ActionSafetyPolicy
  ↓
危険なら停止 / 要レビューなら reviewer へ
  ↓
AgentPermissionGate
  ↓
AI定義の権限確認
  ↓
専門AIを1回だけ起動
  ↓
AgentPermissionGate
  ↓
AI出力を表示前に検査
  ↓
executor
  ↓
操作直前に構造・パス・対象範囲を再検査
```

## 監査ログ

安全判定は `runtime/security/safety_audit.sqlite3` に保存します。

保存するもの:

- 判定時刻
- 入力/出力/AI定義などの scope
- 許可/レビュー要否
- 理由
- リスク分類
- 対象テキストの SHA-256
- 短い伏字プレビュー

全文は保存しません。パスワード、token、API key、メールアドレス、長いランダム文字列っぽいものはプレビュー上で伏字にします。

## ファイル整理 executor の制限

- ファイル本文は読まない。
- 隠しファイルは整理対象にしない。
- 絶対パス、`..`、ドライブ名を拒否する。
- 対象フォルダ外への移動を拒否する。
- Windows禁止文字、予約名、末尾のドット/空白、制御文字を拒否する。
- `.light_ai_logs` への操作を拒否する。
- 削除操作は実装しない。
- 取り消しは直前操作の逆移動と、空の作成フォルダ削除だけに限定する。

## まだ自動化しない操作

以下は v1 では実行器を作らず、専門AIが手順案やレビューだけを返します。

- Webフォーム送信
- ログイン
- 購入/決済
- GitHub push / PR作成 / Release作成
- Windows設定変更
- 管理者権限操作
- 外部サービスへの投稿

将来 executor を追加する場合も、`safety` メタデータ、`ActionSafetyPolicy`、`AgentPermissionGate`、人間確認、監査ログを通す前提にします。
